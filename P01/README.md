# P01: GitHub. Treballant de forma col·laborativa: forks i pull request

## Breu descripció
**Objectiu:** Aprendre a treballar de manera col·laborativa amb GitHub utilitzant **forks** i **pull requests**, simulant el flux de treball professional d’EverPia.

---

## Introducció
A EverPia, no es treballa directament sobre la branca `main` ni s’envien arxius per correu. El flux de treball és:
- Jerarquia d’equip (Team Leader i Membres).
- Revisió estricta de codi mitjançant **Pull Requests**.
- Ús de **forks** per incorporar-se a projectes.

**Escenari:**  
Simularem el procés d’Onboarding d’un nou consultor i configurarem el repositori central de l’empresa.

---

## Flux de treball GitHub a EverPia
1. Cada grup designa un **Team Leader**.
2. La resta són **Membres de l’Equip**.
3. Es treballa sobre el repositori base:
   - **Repositori Base (EverPia-Core):**  
     https://github.com/SMX2m/EverPia-Onboarding

---

## Passos principals
- Fer **fork** del repositori base.
- Clonar el fork localment.
- Crear una branca per a la tasca.
- Fer els canvis necessaris.
- Pujar els canvis al fork.
- Crear un **Pull Request** cap al repositori original.
- Esperar revisió i aprovació.

---

## Objectius específics
- Dominar el flux de treball col·laboratiu amb GitHub.
- Entendre el procés d’integració de canvis mitjançant Pull Requests.
